# Demo-trading TradeClaw US-stock signals on IBKR Paper

**Status:** Plan only. No code, no migrations, no env changes.
**Goal:** Validate the published `hmm-top3` track record (+918%, Sharpe 16, 46% win, +0.32R, 801 sized trades) on the US-stock subset against IBKR Paper, which is the only retail demo venue with real exchange semantics for US equities.
**Author:** Zaky / 2026-05-06.

## 0. Scope and non-goals

- In scope: bridging existing `signal_history` rows (strategy_id = `hmm-top3`, US-stock pair only) into IBKR Paper bracket orders.
- Out of scope: changing signal generation, modifying `apps/web/lib/execution/` (Binance), schema changes, deploying anything to Railway, real-money trading.
- Existing reference contract: `apps/web/lib/execution/executor.ts` and `binance-futures.ts`. The IBKR bridge mirrors its public surface (entry + SL + TP, idempotent via `clientOrderId = signal_id`, persists to `executions` table from migration `018_pilot_executions.sql`).

## 1. Architectural choice

IBKR has no plain REST for retail. Two viable options:

| | Choice A: ib_insync via TWS / IB Gateway | Choice B: IBKR Web API (Client Portal Gateway) |
|---|---|---|
| Transport | TCP socket localhost:7497 (paper) | HTTPS via local Java gateway |
| Maturity | Very mature, large community | Newer, fewer Python wrappers |
| Friction | TWS auto-logout 23:00 ET (Gateway has "no auto-logout") | Daily browser-based reauth, OAuth-like flow |
| Cloud-friendly | No, localhost only | Yes (still needs Java gateway nearby) |
| Best for | Fastest demo trial, single-machine | Future production path |

**Recommendation: Choice A for the 30-day demo trial.** Zaky already has Python in the stack (`scripts/scanner-engine.py`), `ib_insync` is the de facto retail standard, and the bridge runs locally next to TWS. Revisit Choice B only if the demo passes acceptance and we want to move toward a hosted execution layer.

## 2. IBKR paper account setup

1. Sign up at `interactivebrokers.com` (Open Account). Select "Individual" and the country of residence. Paper account is auto-issued for new clients alongside the live account.
2. Existing live clients: Account Management -> Settings -> Paper Trading Account -> Request. Paper account is provisioned within ~24h with a separate username (live username + suffix) and a password you set.
3. Download Trader Workstation (TWS) **or** IB Gateway. Use IB Gateway for the bridge — lighter, headless-ish, and has a "no auto-logout" toggle that TWS lacks.
4. Launch the gateway. Login mode: **Paper Trading**. Note the API socket port — paper defaults to **7497** (TWS) or **4002** (IB Gateway). Live ports are 7496 / 4001 — never use those.
5. In the gateway window: Configure -> Settings -> API -> Settings.
   - Tick "Enable ActiveX and Socket Clients".
   - Untick "Read-Only API" (we need to place orders).
   - Set "Socket port" to 7497 (or 4002 for Gateway).
   - Add `127.0.0.1` to "Trusted IPs".
   - Tick "Allow connections from localhost only" if available.
6. Configure -> Settings -> Lock and Exit -> set "Auto restart" with daily restart at a time the bridge tolerates (e.g. 03:00 local). Gateway can also be set to "never logout" via the same panel.
7. Subscribe to free paper market data: Account Management -> Settings -> Market Data Subscriptions -> US Securities Snapshot and Futures Value Bundle (free for paper). Without this, US equity quotes are delayed/blank and IOC orders mis-price.
8. Verify connection: open a Python REPL on the same machine, run `from ib_insync import IB; ib = IB(); ib.connect('127.0.0.1', 7497, clientId=999); print(ib.isConnected())`. Expect `True`.

## 3. Architecture

```
                 [Postgres signal_history (Railway)]
                            |
                            | poll every 30s, read-only
                            | filter: strategy_id='hmm-top3'
                            |         AND pair IN us_stock_allowlist
                            |         AND created_at > now() - interval '5 min'
                            |         AND NOT EXISTS (executions WHERE signal_id = sh.id)
                            v
              [bridge.py running on Zaky's machine]
                            |
                            | ib_insync over TCP 127.0.0.1:7497
                            v
                  [IBKR IB Gateway Paper @ localhost]
                            |
                            v
                  [IBKR paper exchange simulator]
                            |
                            | (later, optional) write back to executions table
                            v
                 [Postgres executions (Railway, mode='paper-ibkr')]
```

Data flow notes:
- The bridge is read-mostly against `signal_history`. The only write is to `executions` (and `execution_errors` on failure), mirroring the Binance executor's persistence contract from `executor.ts`.
- `clientOrderId = signal_id` for the entry, plus deterministic suffixes for the SL / TP1 children (the Binance executor already uses this pattern via `client-ids.ts` — copy that scheme verbatim so the two brokers don't collide if they ever run side-by-side).
- Postgres `mode` column is set to `paper-ibkr` so the existing `executions` table can hold both Binance and IBKR rows without schema change.

## 4. Symbol coverage (US-stock subset)

Discovery query against `signal_history`:

```sql
SELECT DISTINCT pair
FROM signal_history
WHERE strategy_id = 'hmm-top3'
  AND pair !~ '(USDT|USD|JPY|GBP|CAD|CHF|AUD|NZD)$'
  AND pair NOT IN ('XAUUSD', 'XAGUSD', 'WTIUSD', 'BTCUSD', 'ETHUSD')
ORDER BY pair;
```

This yields equity-shaped tickers (e.g. `AAPL`, `NVDA`, `TSLA`). For each, build:

```python
Stock(symbol='AAPL', exchange='SMART', currency='USD', primaryExchange='NASDAQ')
```

Mapping rules:
- US common stock -> `Stock(SYM, 'SMART', 'USD')`. SMART routing handles venue selection.
- ETFs (SPY, QQQ, etc.) -> same form, `Stock`. They work identically through SMART.
- ADRs (e.g. BABA, TSM) -> same form. Watch for thin liquidity.
- LSE / non-US listings -> reject; out of scope for this demo. Flag any pair where `ib.qualifyContracts()` returns no match or returns a non-USD currency.

Build a hand-curated `us_stock_allowlist.json` (committed to `scripts/demo-trading/ibkr/`) with the resolved contract per pair. The bridge refuses to trade pairs not in the allowlist.

## 5. Bridge sketch (≤50 lines)

Pseudocode for `scripts/demo-trading/ibkr/bridge.py`. **Do not create this file as part of this plan.**

```python
import os, time, json, psycopg2
from ib_insync import IB, Stock, MarketOrder, StopOrder, LimitOrder, Order

ALLOWLIST = json.load(open('us_stock_allowlist.json'))  # {"AAPL": {"primaryExchange":"NASDAQ"}, ...}
PAPER_BALANCE = 1_000_000
RISK_PCT = 0.01  # 1% per trade

def fetch_pending(cur):
    cur.execute("""
      SELECT sh.id, sh.pair, sh.direction, sh.entry, sh.stop_loss, sh.take_profit_1
      FROM signal_history sh
      LEFT JOIN executions e ON e.signal_id = sh.id
      WHERE sh.strategy_id = 'hmm-top3'
        AND sh.created_at > now() - interval '5 minutes'
        AND sh.pair = ANY(%s)
        AND e.signal_id IS NULL
    """, (list(ALLOWLIST.keys()),))
    return cur.fetchall()

def size_shares(entry, stop):
    risk_usd = PAPER_BALANCE * RISK_PCT
    per_share = abs(entry - stop)
    return max(1, int(risk_usd // per_share))  # whole shares only

def place_bracket(ib, sig):
    sid, sym, side, entry, sl, tp = sig
    qty = size_shares(entry, sl)
    contract = Stock(sym, 'SMART', 'USD', primaryExchange=ALLOWLIST[sym]['primaryExchange'])
    action = 'BUY' if side == 'long' else 'SELL'
    parent  = MarketOrder(action, qty); parent.orderId = ib.client.getReqId()
    parent.orderRef = f'tcw_{sid}_entry'; parent.transmit = False
    stop_o  = StopOrder('SELL' if action=='BUY' else 'BUY', qty, sl, parentId=parent.orderId)
    stop_o.orderRef = f'tcw_{sid}_sl'; stop_o.transmit = False
    tp_o    = LimitOrder('SELL' if action=='BUY' else 'BUY', qty // 2, tp, parentId=parent.orderId)
    tp_o.orderRef = f'tcw_{sid}_tp1'; tp_o.transmit = True  # last child transmits all
    for o in (parent, stop_o, tp_o): ib.placeOrder(contract, o)
    return parent, stop_o, tp_o, qty

def main():
    pg = psycopg2.connect(os.environ['DATABASE_URL']); cur = pg.cursor()
    ib = IB(); ib.connect('127.0.0.1', 7497, clientId=42)
    while True:
        for sig in fetch_pending(cur):
            print('placing', sig)
            place_bracket(ib, sig)
            # TODO: INSERT into executions (signal_id, broker='ibkr', mode='paper-ibkr', ...)
        ib.sleep(30)

if __name__ == '__main__': main()
```

Notes:
- Real implementation must add: `executions` insert on success, `execution_errors` insert on failure, qty rounding via `IB.reqContractDetails().minTick`, market-hours guard (skip outside 09:30-16:00 ET unless extended-hours opt-in is added).
- `clientOrderId` lives in `Order.orderRef` for IBKR — IBKR doesn't have a true client-supplied ID like Binance, so `orderRef` is what we assert against for idempotency before placing.
- Idempotency check: before `placeOrder`, call `ib.openTrades()` and skip if any trade has `orderRef.startswith(f'tcw_{sid}_')`.

## 6. Sizing

- Paper account default starting cash: $1,000,000 (configurable in IBKR paper setup; keep at default).
- Risk per trade: 1% = $10,000.
- Position size in shares: `risk_usd / abs(entry - stop)`, then `floor` to whole shares.
- Fractional shares: IBKR supports them on SMART for accounts ≥ $25k, but we deliberately use whole shares only — matches what a small live account would do later and avoids realism distortion.
- Cap per name: hard-cap notional at 25% of equity ($250k) to prevent a tight-stop signal from sucking up the entire account on one trade. Mirror of the Binance executor's `risk-rails.ts` cap pattern.

## 7. Realism check

- IBKR paper has real Level 1 quotes during US RTH (09:30-16:00 ET) provided the free US Securities snapshot bundle is subscribed. Without it, paper market orders fill at delayed-quote midpoints — bad realism.
- Slippage: paper sim is conservative — fills typically inside the visible spread. Live will be worse on illiquid names.
- After-hours fills: paper accepts orders 04:00-20:00 ET if extended-hours flag set, but realism degrades. Keep the bridge RTH-only for the 30-day trial.
- Pattern Day Trader rule: NOT enforced on paper. Live $25k cash equity account WILL hit it on the published cadence — deferred concern, log it and revisit before any real-money attempt.
- Short selling: paper assumes hard-to-borrow availability everywhere. Live will fail on actually-HTB names. Mark all short fills with a `short_realism_flag` in `executions.notes` for later analysis.

## 9. Acceptance criteria

After 30 calendar days of paper trading the US-stock signal subset:

- For each closed paper trade, compute realized R = `(exit_px - entry_px) / abs(entry_px - stop_px) * (1 if long else -1)`.
- Match against the corresponding `signal_history.outcome_24h.pnlPct` (converted to R using the same denominator).
- Pass: realized R is within ±0.15R of the recorded `pnlPct`-derived R for ≥75% of closed trades.
- Aggregate sanity: realized cumulative R within ±10% of `signal_history`-implied cumulative R; win rate within ±5pp of 46%.
- Material divergence (>0.15R) on >25% of trades: stop the bridge, open an issue, investigate causes (slippage, partial fills, halts, after-hours-flag misconfig, missing market data sub).

## 10. Known friction

- **Daily reauth / auto-logout.** TWS forces logout at 23:00 ET. IB Gateway has a "never logout" option — use Gateway. Even Gateway requires weekend manual restart on Sundays.
- **Localhost-only socket.** The bridge cannot run on Railway. It must run on Zaky's machine or a small always-on VM (Mac mini, Hetzner, etc.). This is the single biggest reason to consider Choice B (Web API) later.
- **Market data subscriptions.** Paper market data is separate from live. Without the free US Securities bundle, quotes are delayed/blank and `MarketOrder` fills are unrealistic.
- **No PDT on paper.** Win-rate and trade-frequency stats from this trial OVERSTATE what a $25k live account can do. Real cash-account trial is a separate, later step.
- **Short-sale availability.** Paper assumes infinite borrow. Real shorts on small-caps will fail or get force-bought-in. Tag short fills.
- **Race condition.** If the bridge crashes mid-bracket (parent placed, child not yet placed), the position has no stop until restart. Restart logic must inspect `ib.openTrades()` and re-arm missing children before fetching new signals.
- **Symbol drift.** Tickers change (mergers, ADR delistings). The `us_stock_allowlist.json` must be re-resolved monthly via `ib.qualifyContracts()` and unresolved tickers logged.
